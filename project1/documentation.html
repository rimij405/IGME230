The Project

This page has been updated for Project 3. Rock Paper Scissors using the Pixi.JS library.

Meeting the Requirements

There are many requirements for this project. See below, for how I’ve approached each one.

Use of Structural Tags

By using a modern browser, such as Google Chrome or Mozilla Firefox, you can view the page source, or, inspect a page using their respective developer consoles. From there, you can see a clear use of ‹header›, ‹section›, ‹article›, ‹nav›, and ‹footer›" tags being used appropriately.

Consistent Layout

While the design for each individual page may vary slightly, the overall core design has been maintained and kept alive with a consistent use of C.R.A.P. principles.

CSS and an External Stylesheet

You can gain access to the external stylesheet by using the developer console, or, visiting it here.

Responsive & Accessible

The site itself is guaranteed to be responsive for screens 480px and above. All images have alternative text supplied through the use of the ‹img› element’s alt attribute. Mobile devices and screen-readers have access to a #SkipToContent anchor in the page. The colors used on the site have been chosen for high-contrast, to ensure readability in users with varying levels of eyesight. Images are not resized by CSS; they have been prepared explicitly for the web using an image-editing program.

No Templates

This HTML, CSS, and JavaScript are all developed from scratch.

Critique-Ready and Validated

This project was prepared and ready for critique on Wednesday, September 27th, 2017. The webpage’s HTML and CSS both validate.

Challenges

My biggest challenge was getting used to programming in JavaScript; it's hard to anticipate exactly what each object will have and thus, you're very reliant on the API's that are provided..

The Process

I spent a majority of my time engaging with the content of the pages, to ensure I had generated all of the content that I would need, in advance, for my website. Writing all of my content out, in advance, meant I could focus solely on the design of the front-facing pages without having to worry about serious changes down the line.

After content was locked-down and copyedited, I began the design process by sketching out plans onto graph paper. After about three passes, I was left with various takes on navigation styles, as well as notes on media queries for my support of mobile devices. Using a placeholder website, I was able to block out the elements of the webpage without worrying about having the final art assets in place.

After the critique, I focused on polishing the website.
